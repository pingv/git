Git Command Guide

Hover over each part of the command to see what it does.

1. Basic Operations

Initialize Repository:

git Version control system command
init Initialize a new Git repository

Stage Changes:

git Version control system command
add Add files to staging area
. Stage all files in current directory

Commit Changes:

git Version control system command
commit Record changes to the repository
-m Provide commit message
"message" Your descriptive commit message

2. Branch Management

List Branches:

git Version control system command
branch List, create, or delete branches
-a Show all branches (local and remote)

Create New Branch:

git Version control system command
checkout Switch branches or restore files
-b Create and switch to new branch
feature/new-feature Name of your new branch

3. Remote Operations

Clone Repository:

git Version control system command
clone Copy a repository from remote source
URL Remote repository URL

Push Changes:

git Version control system command
push Upload local changes to remote
origin Remote repository name
main Branch name to push to

4. Advanced Operations

Interactive Rebase:

git Version control system command
rebase Reapply commits on top of another base
-i Interactive mode
HEAD~3 Last three commits

Reset Hard:

git Version control system command
reset Reset current HEAD to specified state
--hard Discard all changes in working directory
origin/main Reset to remote main branch
